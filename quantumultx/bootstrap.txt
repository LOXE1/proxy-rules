Quantumult X 一次性接入说明（稳定版）

本文件不是完整配置，不包含机场订阅地址、节点、密码或证书。
应用规则从 GitHub 每 86400 秒自动检查更新。

一、策略组

保留以下应用策略：

- OpenAI
- Claude
- Copilot
- TikTok
- 电报代理
- Discord

删除或忽略旧的 Gemini 独立策略组。Gemini、Google 与 YouTube 全部使用订阅原始规则，不再做专属分流。

Discord 策略示例：

static = Discord, 狮城节点, 香港节点, 日本节点, 美国节点, 台湾节点, 手动选择, 节点选择

二、清理 [filter_local]

删除旧配置中手工写入的 OpenAI、Gemini、Claude、Copilot、Discord、Telegram、TikTok 应用规则，避免与远程规则重复。

必须删除所有指向 Gemini 的 Google/YouTube 公共规则，包括但不限于：

host,gemini.google.com,Gemini
host,www.google.com,Gemini
host,accounts.google.com,Gemini
host,www.youtube.com,Gemini
host,i.ytimg.com,Gemini
host,yt3.ggpht.com,Gemini
host-suffix,googleapis.com,Gemini
host-suffix,gstatic.com,Gemini

[filter_local] 建议只保留必要的本地优先规则，例如：

GEOIP,CN,direct
FINAL,最终分流

三、修改 [filter_remote]

删除或禁用旧的重复应用规则资源：

- OpenAI
- Claude
- Gemini
- Copilot
- TikTok
- Telegram
- Discord

在广告规则之后、苹果与媒体规则之前加入：

https://raw.githubusercontent.com/LOXE1/proxy-rules/main/quantumultx/managed.list, tag=LOXE1自动分流, update-interval=86400, opt-parser=false, enabled=true

继续保留：

- 苹果服务
- 亚洲媒体
- 国内媒体
- 国际媒体
- 局域网直连
- 中国大陆网站直连
- 国外网站代理

四、Google / Gemini / YouTube

不要加载 Gemini 或 YouTube 专属远程规则。

它们继续使用原有 Google、国际媒体或国外网站规则，保证同一浏览会话尽量保持同一出口。

不要全局丢弃 QUIC；删除或注释：

udp_drop_list = QUIC

五、首次刷新

保存后在 Quantumult X 的资源页面手动更新一次“LOXE1自动分流”，再完全重启 Quantumult X。

以后 `managed.list` 更新后会按 `update-interval` 自动同步。
